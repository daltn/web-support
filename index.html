Command Line Fundamentals

The command line interface (CLI) is a text-based way to interact with your computer. While it might seem intimidating at first, mastering the command line will make you significantly more efficient as a developer and give you access to powerful tools that don't have graphical interfaces.

Why Learn Command Line?

Modern web development relies heavily on command line tools. Package managers, build tools, version control systems, and deployment scripts all operate through the terminal. Learning the command line is essential for:

  • Installing and managing development tools
  • Navigating file systems efficiently
  • Using Git for version control
  • Running build processes and scripts
  • Deploying applications to servers

Understanding Your Terminal

The terminal is an application that provides access to the command line. On macOS, it's called "Terminal." On Windows, you can use "Command Prompt," "PowerShell," or install "Git Bash" for a Unix-like experience.

The Command Prompt

When you open a terminal, you'll see a prompt that typically shows your username, computer name, and current directory:

username@computer:~/current-directory $

Essential Navigation Commands

pwd - Print Working Directory

Shows your current location in the file system:

pwd
/Users/yourname/Documents

ls - List Directory Contents

Displays files and folders in the current directory:

ls
Desktop Documents Downloads Pictures Music

Use flags to modify the output:

ls -la
drwxr-xr-x 5 user staff 160 Oct 15 09:30 Desktop drwxr-xr-x 8 user staff 256 Oct 15 10:15 Documents drwxr-xr-x 3 user staff 96 Oct 15 08:45 Downloads

Command Flags Explained

-l: Long format (shows permissions, size, date)
-a: Show hidden files (starting with .)
-h: Human readable file sizes

cd - Change Directory

Navigate between folders:

cd Documents
cd ..

.. means "parent directory" (go up one level)

cd ~

~ means "home directory" (your user folder)

File and Folder Management

mkdir - Make Directory

Create new folders:

mkdir my-new-folder
mkdir -p projects/web-development/portfolio

The -p flag creates parent directories if they don't exist.

touch - Create Files

Create empty files:

touch index.html
touch style.css script.js

cp - Copy Files and Folders

cp file.txt backup.txt
cp -r folder-name folder-copy

The -r flag copies directories recursively.

mv - Move and Rename

mv old-name.txt new-name.txt
mv file.txt different-folder/

rm - Remove Files and Folders

rm unwanted-file.txt
rm -rf unwanted-folder

Warning: rm permanently deletes files. Use with caution!

Getting Help

Most commands have built-in help:

The man command shows the manual page for any command.

Practice Makes Perfect

The command line becomes natural with practice. Try to use it for daily file operations instead of the graphical file manager. Start with simple tasks and gradually incorporate more commands into your workflow.

Git & GitHub

Git is a distributed version control system that tracks changes in your files and enables collaboration with other developers. GitHub is a cloud-based platform that hosts Git repositories and provides additional collaboration features.

What is Version Control?

Version control is like having an unlimited undo button for your entire project. It tracks every change you make, allows you to revert to previous versions, and enables multiple people to work on the same project without conflicts.

Why Git is Essential

  • Track Changes: See exactly what changed, when, and who changed it
  • Backup: Your code is safely stored and retrievable
  • Branching: Work on features without breaking the main code
  • Collaboration: Multiple developers can work simultaneously
  • Deployment: Many hosting services integrate directly with Git

Git Fundamentals

Repository (Repo)

A repository is a project folder that Git is tracking. It contains all your files plus a hidden .git folder that stores the version history.

Working Directory, Staging Area, and Repository

Git has three main areas:

  • Working Directory: Your actual files that you edit
  • Staging Area: Files prepared for the next commit
  • Repository: Committed snapshots of your project

Installing Git

Download Git from git-scm.com and follow the installation instructions for your operating system.

First-Time Setup

Configure Git with your name and email:

git config --global user.name "Your Name" git config --global user.email "your.email@example.com" # Check your configuration git config --list

Basic Git Commands

git init - Initialize Repository

Turn any folder into a Git repository:

cd collage-portfolio git init

git status - Check Repository Status

See which files have changed and what's staged:

git status
On branch main No commits yet Untracked files: (use "git add ..." to include in what will be committed) css/ index.html js/ nothing added to commit but untracked files present

git add - Stage Files

Add files to the staging area:

git add index.html
git add css/style.css
git add .

The dot (.) adds all changed files in the current directory.

git commit - Save Changes

Create a snapshot of your staged changes:

git commit -m "Initial project setup with basic file structure"

Writing Good Commit Messages

Use clear, descriptive messages in present tense:
✅ "Add responsive navigation menu"
✅ "Fix image loading bug in gallery"
❌ "Updated stuff"
❌ "Fixed things"

git log - View Commit History

git log
git log --oneline
commit 2f8d1a3b... (HEAD -> main) Author: Your Name Date: Thu Oct 15 10:30:00 2025 -0700 Initial project setup with basic file structure

GitHub Integration

What is GitHub?

GitHub is a cloud platform that hosts Git repositories and adds collaboration features like issue tracking, pull requests, and project management tools.

Creating a GitHub Account

  1. Go to github.com and sign up
  2. Choose a username that represents you professionally
  3. Verify your email address

Creating a Repository on GitHub

  1. Click the "+" icon in the top right
  2. Select "New repository"
  3. Name it "collage-portfolio"
  4. Make it public (so it can be hosted for free)
  5. Don't initialize with README (we already have files)
  6. Click "Create repository"

Connecting Local Repository to GitHub

# Add GitHub as remote origin git remote add origin https://github.com/yourusername/collage-portfolio.git # Verify the remote was added git remote -v # Push your code to GitHub git branch -M main git push -u origin main

Daily Git Workflow

Making Changes

# Check what's changed git status # Add specific files git add index.html css/style.css # Or add all changes git add . # Commit with a descriptive message git commit -m "Add hero section with background image" # Push to GitHub git push

Pulling Changes

If you're working from multiple computers or collaborating:

git pull

Branching

Branches allow you to work on features without affecting the main code:

# Create and switch to new branch git checkout -b feature/gallery-lightbox # Work on your feature, make commits git add . git commit -m "Add lightbox functionality to gallery" # Switch back to main branch git checkout main # Merge your feature git merge feature/gallery-lightbox # Delete the feature branch git branch -d feature/gallery-lightbox

GitHub Pages

GitHub can host simple websites for free:

  1. Go to your repository settings
  2. Scroll to "Pages" section
  3. Select "Deploy from a branch"
  4. Choose "main" branch
  5. Your site will be available at username.github.io/repository-name

Git Best Practices

• Commit often with small, logical changes
• Write clear commit messages
• Pull before pushing to avoid conflicts
• Use branches for new features
• Never commit sensitive information (passwords, API keys)

Common Git Scenarios

Undoing the Last Commit

git reset --soft HEAD~1

This undoes the commit but keeps your changes staged.

Fixing a Commit Message

git commit --amend -m "Corrected commit message"

Seeing What Changed

git show

Shows the last commit's changes.

HTML Structure

HTML (HyperText Markup Language) is the foundation of all web pages. It provides structure and meaning to content, telling browsers how to organize and display information. Modern HTML5 emphasizes semantic markup that's accessible to all users and search engine friendly.

What is HTML?

HTML uses elements (tags) to mark up content. Each element has a specific purpose and meaning, from headings and paragraphs to navigation and articles. Think of HTML as the skeleton of your webpage—it provides structure that CSS will style and JavaScript will make interactive.

HTML Document Structure

Every HTML document follows the same basic structure:

<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>Page Title</title> </head> <body> <!-- Your content goes here --> </body> </html>

Essential HTML Elements

Document Declaration

<!DOCTYPE html>

Tells the browser this is an HTML5 document.

Root Element

<html lang="en">

Contains all content. The lang attribute helps screen readers and translation tools.

Head Section

Contains metadata about the document:

<head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <meta name="description" content="Digital collage portfolio by [Your Name]"> <title>[Your Name] - Digital Collage Artist</title> <link rel="stylesheet" href="css/style.css"> </head>

Semantic HTML5 Elements

Semantic elements clearly describe their purpose to both browsers and developers:

<header>

Contains introductory content, typically navigation and site branding:

<header> <nav> <h1>Your Name</h1> <ul> <li><a href="#about">About</a></li> <li><a href="#gallery">Gallery</a></li> <li><a href="#contact">Contact</a></li> </ul> </nav> </header>

<main>

Contains the primary content of the page:

<main> <!-- All your main content sections go here --> </main>

<section>

Represents a distinct section of content:

<section id="about"> <h2>About My Work</h2> <p>I create digital collages that blend photography, illustration, and design...</p> </section>

<article>

Represents independent, reusable content:

<article class="gallery-item"> <img src="images/collage-1.jpg" alt="Abstract collage with vibrant colors"> <h3>Urban Dreams</h3> <p>A digital exploration of city life and nature.</p> </article>

<aside>

Contains content related to the main content but not essential:

<aside> <h3>Artist Statement</h3> <p>My work explores the intersection of digital and analog...</p> </aside>

<footer>

Contains footer information:

<footer> <p>© 2025 Your Name. All rights reserved.</p> <p>Contact: <a href="mailto:you@example.com">you@example.com</a></p> </footer>

Important HTML Attributes

alt Attribute for Images

Always provide descriptive alt text for accessibility:

<img src="collage.jpg" alt="Abstract digital collage with blue and orange geometric shapes">

loading="lazy" for Performance

Improves page load speed by loading images only when needed:

<img src="large-image.jpg" alt="Description" loading="lazy">

target="_blank" for External Links

<a href="https://instagram.com/yourusername" target="_blank">Follow on Instagram</a>

Forms and Input Elements

Forms collect user input. Each input should have a label for accessibility:

<form> <div class="form-group"> <label for="name">Name</label> <input type="text" id="name" name="name" required> </div> <div class="form-group"> <label for="email">Email</label> <input type="email" id="email" name="email" required> </div> <div class="form-group"> <label for="message">Message</label> <textarea id="message" name="message" rows="5"></textarea> </div> <button type="submit">Send</button> </form>

HTML Validation and Best Practices

HTML Best Practices

• Use semantic elements for better accessibility
• Always include alt attributes on images
• Use proper heading hierarchy (h1, h2, h3...)
• Validate your HTML at validator.w3.org
• Keep your HTML clean and properly indented

Accessibility Considerations

  • Semantic HTML: Use elements for their intended purpose
  • Alt Text: Describe images meaningfully
  • Form Labels: Associate labels with form controls
  • Heading Hierarchy: Use h1-h6 in logical order
  • Keyboard Navigation: Ensure all interactive elements are accessible

CSS Styling

CSS (Cascading Style Sheets) transforms your HTML structure into beautiful, responsive websites. It controls layout, colors, typography, animations, and how your site adapts to different screen sizes. Modern CSS provides powerful tools like Grid and Flexbox for creating sophisticated layouts.

What is CSS?

CSS works by selecting HTML elements and applying style rules to them. It follows a cascading system where styles can inherit from parent elements and be overridden by more specific rules. The "cascade" determines which styles take priority when conflicts arise.

CSS Syntax

selector { property: value; another-property: another-value; } /* Example */ h1 { color: #ffffff; font-size: 48px; font-weight: 300; }

CSS Selectors

Selectors target specific HTML elements for styling:

Element Selectors

/* Selects all h1 elements */ h1 { color: #ffffff; } /* Selects all paragraphs */ p { line-height: 1.6; }

Class Selectors

/* Selects elements with class="gallery-item" */ .gallery-item { background: #222; padding: 20px; } /* Multiple classes */ .gallery-item.featured { border: 2px solid #00d4ff; }

ID Selectors

/* Selects element with id="hero" */ #hero { height: 100vh; display: flex; align-items: center; }

Descendant Selectors

/* Selects images inside gallery items */ .gallery-item img { width: 100%; height: auto; } /* Selects links inside navigation */ nav a { text-decoration: none; color: #ccc; }

CSS Reset and Base Styles

Start with a CSS reset to ensure consistent styling across browsers. Create css/reset.css:

/* CSS Reset */ * { margin: 0; padding: 0; box-sizing: border-box; } html { scroll-behavior: smooth; } body { font-family: 'Helvetica Neue', Arial, sans-serif; line-height: 1.6; color: #e5e5e5; background-color: #1a1a1a; } img { max-width: 100%; height: auto; display: block; } a { color: inherit; text-decoration: none; } button { border: none; background: none; font: inherit; cursor: pointer; } ul { list-style: none; }

Typography

Good typography is essential for readable, professional websites:

/* Typography System */ h1 { font-size: 48px; font-weight: 300; color: #ffffff; letter-spacing: -1px; margin-bottom: 20px; } h2 { font-size: 32px; font-weight: 300; color: #ffffff; letter-spacing: -0.5px; margin: 40px 0 20px 0; } h3 { font-size: 24px; font-weight: 400; color: #ffffff; margin: 30px 0 15px 0; } p { font-size: 16px; line-height: 1.6; color: #ccc; margin-bottom: 15px; max-width: 65ch; /* Optimal reading width */ } /* Responsive typography */ @media (max-width: 768px) { h1 { font-size: 36px; } h2 { font-size: 28px; } h3 { font-size: 20px; } }

Layout with Flexbox

Flexbox is perfect for one-dimensional layouts like navigation bars and centering content:

/* Navigation Layout */ nav { display: flex; justify-content: space-between; align-items: center; padding: 20px 40px; background: #111; } .nav-links { display: flex; gap: 30px; } /* Hero Section Centering */ #hero { display: flex; flex-direction: column; justify-content: center; align-items: center; height: 100vh; text-align: center; } /* Flexible Content Layout */ .about-content { display: flex; gap: 40px; align-items: center; } .about-text { flex: 1; } .about-image { flex: 0 0 300px; } /* Responsive Flexbox */ @media (max-width: 768px) { .about-content { flex-direction: column; } nav { flex-direction: column; gap: 20px; } }

Layout with CSS Grid

Grid is ideal for two-dimensional layouts like image galleries:

/* Gallery Grid */ .gallery-grid { display: grid; grid-template-columns: repeat(auto-fit, minmax(300px, 1fr)); gap: 30px; padding: 40px 0; } /* Responsive Grid Variations */ @media (min-width: 1200px) { .gallery-grid { grid-template-columns: repeat(3, 1fr); } } @media (max-width: 768px) { .gallery-grid { grid-template-columns: 1fr; gap: 20px; } } /* Process Steps Grid */ .process-steps { display: grid; grid-template-columns: repeat(auto-fit, minmax(250px, 1fr)); gap: 30px; margin-top: 40px; }

Why Image Optimization Matters

Images typically account for 60-70% of a website's total size. Unoptimized images lead to slow loading times, poor user experience, and higher bandwidth costs. For a collage portfolio, proper image optimization is essential for showcasing your work professionally.

Common Image Problems

  • Poor loading: All images loading at once instead of as needed

Understanding Cloudinary URLs

Cloudinary URLs follow a specific structure that enables transformations:

  • [CLOUD_NAME]: Your unique identifier
  • image/upload: Specifies image delivery
  • [TRANSFORMATIONS]: Size, format, quality parameters
  • [IMAGE_NAME]: Your original filename
